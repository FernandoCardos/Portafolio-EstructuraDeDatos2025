La actividad consistió en la implementación práctica de nuestro algoritmo de ordenamiento asignado (ej. Cocktail Sort) dentro de un repositorio de GitHub compartido. El objetivo era completar las funciones vacías del archivo base sorting.c con la lógica de nuestro algoritmo.

El flujo de trabajo se centró en usar Git: primero, clonamos el repositorio base, luego creamos una rama por equipo (ej. E03-cocktail) para trabajar de forma aislada. Después de implementar y probar la función, hicimos commits con mensajes claros y generamos un Pull Request (PR) hacia la rama principal. Esto nos obligó a seguir un proceso de desarrollo real.

Reflexión Personal

Lo que Aprendí: El mayor aprendizaje fue el flujo de trabajo real de Git y GitHub (clonar ->branch -> commit ->PR). Fue la primera vez que integramos código a una rama principal con un PR, y eso fue crucial.

Problemas: El reto no solo fue traducir la teoría del algoritmo a código C, sino también asegurarse de que la integración vía Pull Request no generara conflictos con el código base.
