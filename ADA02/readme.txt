La tarea consistió en programar un sistema de simulación para un banco que tiene tres cajas de atención. El programa generó clientes aleatorios con un número variable de transacciones. El reto principal fue la asignación aleatoria de cada cliente a una de las tres colas disponibles.

Tuvimos que registrar todo el proceso (entrada a la cola, qué cola, y salida de la caja). El objetivo final era generar estadísticas clave, como el tiempo promedio de espera por cada cola y el total de clientes atendidos, para entender la eficiencia del sistema.

Reflexión Personal

Lo que Aprendí: Pude aplicar la estructura de datos Cola (Queue) de manera profunda y a generar procesos y tiempos de forma aleatoria, algo fundamental en simulación.

Problemas: El desafío fue asegurar la sincronización entre la llegada de un cliente y la disponibilidad de las tres cajas para que el cálculo del tiempo de espera fuera preciso.
